# Book Verify

> Verify HTML output from the bookSHelf pipeline against a comprehensive quality checklist covering document structure, security (no polyfill.io), navigation elements, content CSS classes, solution expandables, anchor IDs, images, and template placeholders. Reports PASS/FAIL/WARNINGS with specific line numbers.

## Prerequisites
- HTML file generated by book-html-gen
- File located in `projects/{project}/html/` or `docs/` directory

## When to Use
- After book-html-gen produces HTML output
- Before book-publish to ensure quality
- User requests verification of any bookSHelf HTML file
- After manual edits to HTML files

## When NOT to Use
- Verifying markdown content (use book-math-verify instead)
- Verifying mapping JSON or merge output (manual review)
- Content hasn't been converted to HTML yet

## Guardrails

> ⚠️ **Must NOT:**
> - Auto-fix issues without reporting them first
> - Skip the security check (polyfill.io is CRITICAL)
> - Report PASS if any CRITICAL check fails
> - Modify the HTML file during verification (report only)

## Quick Start
1. Read the HTML file
2. Run through all checklist items
3. Report results as PASS/FAIL/WARNINGS with line numbers

## Workflow

### Phase 1: Security Check (CRITICAL - First)
- **INPUT:** HTML file path
- **ACTION:** Check for security issues FIRST:
```bash
grep -n "polyfill.io" "{file.html}"
```
  - If polyfill.io found: IMMEDIATE FAIL, stop and report
- **OUTPUT:** Security check pass/fail

### Phase 2: Structure Check
- **INPUT:** HTML file
- **ACTION:** Verify document structure:
  - Valid HTML5 doctype
  - `<html lang="en">` present
  - `<meta charset="UTF-8">` in head
  - `<meta name="viewport">` in head
  - `<title>` tag with section name
  - MathJax script from cdn.jsdelivr.net
- **OUTPUT:** Structure check results

### Phase 3: Navigation Check
- **INPUT:** HTML file
- **ACTION:** Verify navigation elements:
  - `<nav class="navbar">` with sticky positioning
  - TOC dropdown
  - Sections/Examples/Problems/Videos dropdowns
  - Dark mode toggle `id="themeToggle"`
  - Floating nav buttons `.float-nav`
  - H1 title wrapped in home link to `../index.html`
- **OUTPUT:** Navigation check results

### Phase 4: Content Classes Check
- **INPUT:** HTML file
- **ACTION:** Verify all content elements use correct CSS classes:
  - `.learning-objectives`, `.context-pause`, `.insight-note`
  - `.definition` with `.definition-title`
  - `.example` with `.example-title`
  - `.important-rule`, `.problem-set`, `.problem`
- **OUTPUT:** Content class check results

### Phase 5: Solution Expandables Check (CRITICAL)
- **INPUT:** HTML file
- **ACTION:** Verify ALL `<details>` elements have `class="solution"`:
```bash
grep -n "<details>" "{file.html}" | grep -v 'class="solution"'
```
  - Each `<details class="solution">` must contain `<summary>`
  - Solution content in `<div class="solution-content">`
- **OUTPUT:** Solution expandables check results

### Phase 6: Anchors and Images Check
- **INPUT:** HTML file
- **ACTION:**
  - H2 headers have id attributes for TOC linking
  - Example/problem divs have matching id attributes
  - TOC links match actual element IDs
  - All `<img>` have src and alt attributes
  - Image paths are relative
- **OUTPUT:** Anchor and image check results

### Phase 7: Report
- **INPUT:** All check results
- **ACTION:** Produce structured report:
```markdown
## HTML Verification: [filename]

### PASS ✅
- [items that passed]

### FAIL ❌
- [items that failed with line numbers]

### WARNINGS ⚠️
- [items that are concerning but not blocking]
```
- **OUTPUT:** Complete verification report

## Error Handling

| Problem | Action |
|---------|--------|
| polyfill.io found | CRITICAL fail - must be removed before publish |
| Missing solution class | List all affected line numbers for batch fix |
| Broken anchor links | List mismatched TOC link → element ID pairs |
| Missing images | List image paths that don't resolve |

## Common Mistakes

| Mistake | Fix |
|---------|-----|
| Skipping polyfill.io check | ALWAYS check this first - it's a security issue |
| Not checking every `<details>` element | Use grep to find ALL details without class="solution" |
| Missing line numbers in report | Always include line numbers for failures |
| Reporting PASS with warnings | PASS means all critical checks passed; warnings are separate |
